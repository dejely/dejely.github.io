privmotion

Privacy-preserving motion analytics for videos, RGB-D streams, images, and frame folders.

Overview

privmotion turns identity-bearing visual input into inspectable motion artifacts while avoiding raw RGB retention by default. It is designed as a research prototype for studying kinematics without making source footage the primary output.

Key Capabilities

  • Exports skeleton keypoints, silhouettes, depth-like surrogates, and motion features.
  • Supports image files, videos, RGB-D streams, and directories of frames.
  • Provides YOLO-Pose support with a deterministic prototype fallback.
  • Builds previews and benchmark reports from anonymized outputs.
  • Includes optional feature encryption, access policies, and audit records.

Technical Approach

The Python pipeline processes RGB frames in memory, writes machine-readable kinematic outputs, and records backend and retention metadata for validation. An OpenCV-style C++ and CMake scaffold defines a path toward a future native module.

This is a research prototype, not an official OpenCV module or a production privacy guarantee. Its HIPAA-oriented aggregate mode supports qualified expert review but does not independently certify HIPAA de-identification.

Stack

Python, OpenCV, YOLO Pose, NumPy, C++, CMake, pytest, and imageio.
